# Poker Timer — Settings Screen Design Spec
**Date:** 2026-04-04

## Context

Расширение существующего `poker-timer.html`. Добавляем экран настроек, позволяющий редактировать структуру блайндов и временны́е параметры. Всё остаётся в одном файле.

---

## Architecture

### Новые данные

```js
const DEFAULT_CONFIG = {
  levelDuration: 20,   // минуты
  breakDuration: 10,   // минуты
  breakEvery: 2,       // каждые N уровней
  blindLevels: [
    { sb: 10,  bb: 20   },
    { sb: 20,  bb: 40   },
    // ... все 14 уровней
  ],
};
```

`config` — активная конфигурация, загружается из `localStorage` (ключ `pokerTimerConfig`) при старте. Если записи нет — используется `DEFAULT_CONFIG`.

### Изменения существующего кода

- `BLIND_LEVELS`, `LEVEL_DURATION`, `BREAK_DURATION`, `BREAK_EVERY` становятся переменными (`let`), инициализируются из `config` при загрузке
- `buildStages()` остаётся без изменений, но вызывается повторно через `rebuildStages()` при применении настроек
- `rebuildStages()` пересобирает `stages`, сбрасывает `currentStage = 0`, `timeLeft = stages[0].duration`, `warnedOneMin = false`
- `stages` становится `let` вместо `const`

### Переключение экранов

CSS-класс на `body`:
- нет класса → таймер виден, настройки скрыты
- `body.is-settings` → настройки видны, таймер скрыт

```css
#settings-screen { display: none; }
body.is-settings #settings-screen { display: flex; flex-direction: column; height: 100vh; }
body.is-settings .top,
body.is-settings .timer-wrap,
body.is-settings .progress-wrap,
body.is-settings .bottom { display: none; }
```

## Break Dividers в таблице

Разделители `── ☕ Перерыв ──` вставляются динамически после каждой N-й строки (N = значение поля «каждые»). Обновляются при:
- изменении поля «каждые N уровней»
- добавлении/удалении строки

Реализация: при каждом изменении полностью перерисовывать таблицу (`renderBlindsTable()`). Разделитель — строка `<tr class="break-divider">` с colspan=4.

## Кнопки настроек

### ⚙ (правый верхний угол, сейчас скрыт)
Активировать: убрать `style="visibility:hidden;"`. Добавить `onclick="openSettings()"`.

### `openSettings()`
- Ставит таймер на паузу если работал
- Добавляет `body.is-settings`
- Рендерит форму настроек из текущего `config`

### `closeSettings()` (кнопка «← Назад»)
- Убирает `body.is-settings`
- Никаких изменений в `config` или `stages`

### `saveSettings()`  (кнопка «Сохранить»)

1. Читает значения из формы, валидирует:
   - `levelDuration` и `breakDuration` — целые числа > 0
   - `breakEvery` — целое число > 0
   - `blindLevels` — минимум 1 уровень, все SB и BB — положительные целые числа
2. Если валидация не прошла — показывает сообщение об ошибке рядом с полем, не закрывает
3. Если игра активна (не на паузе или `currentStage > 0` и `timeLeft < stages[0].duration`) — вызывает `confirm('Изменение настроек сбросит текущую игру. Продолжить?')`
4. Если подтверждено (или игра не активна) — применяет настройки:
   - Обновляет `config`
   - Сохраняет в `localStorage`
   - Вызывает `rebuildStages()`
   - Вызывает `restartTournament()`
   - Закрывает настройки

### `resetToDefaults()` (кнопка «сбросить к умолчаниям»)
- Перезаполняет форму значениями из `DEFAULT_CONFIG` (не сохраняет в localStorage — только форму)

## Валидация

| Поле | Правило | Сообщение об ошибке |
|------|---------|---------------------|
| Длительность уровня | целое число 1–999 | «Введите целое число от 1 до 999» |
| Перерыв | целое число 1–999 | «Введите целое число от 1 до 999» |
| Каждые N уровней | целое число ≥ 1 | «Введите целое число ≥ 1» |
| SB/BB каждого уровня | целое число > 0 | Подсветка строки красным |
| Количество уровней | ≥ 1 | «Добавьте хотя бы один уровень» |

## localStorage

Ключ: `pokerTimerConfig`
Формат: JSON-строка объекта `config`

```js
// Сохранение
localStorage.setItem('pokerTimerConfig', JSON.stringify(config));

// Загрузка
function loadConfig() {
  try {
    const saved = localStorage.getItem('pokerTimerConfig');
    return saved ? JSON.parse(saved) : structuredClone(DEFAULT_CONFIG);
  } catch {
    return structuredClone(DEFAULT_CONFIG);
  }
}
```

---

## Определение «активной игры»

Игра считается активной если: `currentStage > 0 || timeLeft < stages[0].duration`

Иными словами — если пользователь хоть что-то сделал после старта (перешёл на другой уровень или время частично прошло). Если игра ещё не начиналась (Round 1, 20:00, на паузе) — подтверждения не запрашиваем.
