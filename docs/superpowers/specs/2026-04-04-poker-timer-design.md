# Poker Timer — Design Spec
**Date:** 2026-04-04

## Context

Домашняя игра Texas Hold'em, до 10 игроков. Таймер запускается на ноутбуке с Windows в Chrome. Никакого сервера, никаких зависимостей — один HTML-файл.

---

## Architecture

Единственный файл: `poker-timer.html`

- Открывается двойным кликом в Chrome на Windows
- Весь CSS внутри `<style>`, весь JS внутри `<script>`
- Состояние хранится в памяти (не переживает обновление страницы — приемлемо для MVP)
- Звук через Web Audio API (встроен в браузер, без библиотек)

---

## Blind Structure

14 уровней, захардкожены в JS-массив:

| Уровень | SB   | BB   |
|---------|------|------|
| 1       | 10   | 20   |
| 2       | 20   | 40   |
| 3       | 30   | 60   |
| 4       | 40   | 80   |
| 5       | 60   | 120  |
| 6       | 80   | 160  |
| 7       | 100  | 200  |
| 8       | 150  | 300  |
| 9       | 200  | 400  |
| 10      | 300  | 600  |
| 11      | 400  | 800  |
| 12      | 600  | 1200 |
| 13      | 800  | 1600 |
| 14      | 1000 | 2000 |

**Длительность уровня:** 20 минут
**Перерыв:** 10 минут, автоматически после каждого чётного уровня (после 2, 4, 6, 8, 10, 12, 14)

Внутреннее представление — плоский массив этапов (`stages`), где каждый этап имеет тип `"level"` или `"break"`:

```js
[
  { type: "level", index: 1, sb: 10, bb: 20, duration: 1200 },
  { type: "break", duration: 600 },
  { type: "level", index: 3, sb: 30, bb: 60, duration: 1200 },
  ...
]
```

---

## State

```js
let currentStage = 0;   // индекс в массиве stages
let timeLeft = 1200;    // секунды до конца текущего этапа
let isPaused = true;    // таймер стартует на паузе
let timerId = null;     // setInterval handle
```

Таймер стартует на паузе — пользователь нажимает ▶ сам, когда готов.

---

## UI Layout

```
┌─────────────────────────────────────────────┐
│ [⏱]        Round 3 · 30 / 60          [⚙]  │  ← top bar
│                                             │
│                   18:42                     │  ← giant timer (fills screen)
│                                             │
│ ══════════════════░░░░░░░░░░░░░░░░░░░░░░░  │  ← progress bar
│   Next: ☕ Break 10 min → then 40 / 80      │
│         [⏮]  [⏪]  [⏸]  [⏩]  [⏭]         │  ← controls
└─────────────────────────────────────────────┘
```

**Детали:**

- Фон: `#1a1a1a` (почти чёрный)
- Таймер: белый, `font-size` через `clamp(140px, 22vw, 240px)`, `font-weight: 900`
- Прогресс-бар: тонкая полоска (3px), цвет заполнения — фиолетовый `#7c3aed`; показывает долю **прошедшего** времени этапа
- Во время перерыва: вместо блайндов — «☕ Break», таймер показывает время перерыва
- Кнопки управления: фиолетовый фон для основных, тёмный (`#2a2a2a`) для ⏮ и ⏭

**Кнопки:**

| Кнопка | Действие |
|--------|----------|
| ⏮ | Сброс на начало текущего этапа |
| ⏪ | Перейти на предыдущий этап |
| ⏸/▶ | Пауза / Старт |
| ⏩ | Перейти на следующий этап |
| ⏭ | Перейти в конец (последний уровень) |

---

## Sound

Web Audio API, без внешних файлов.

| Событие | Звук |
|---------|------|
| За 60 секунд до конца уровня | Короткий одиночный beep (предупреждение) |
| Смена уровня / начало перерыва | Двойной beep (переход) |
| Конец перерыва | Тройной beep (возвращение к игре) |

Звук генерируется программно через `OscillatorNode`. Не требует файлов и работает без интернета.

---

## End State

После последнего уровня (14) таймер останавливается, на экране показывается сообщение «Tournament Over».
